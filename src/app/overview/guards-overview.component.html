Guards

A way to protect routes

CanActivate To restrict access or check prerequisites CanActivateChild Same but for all the children CanDeactivate Prevent leaving with incomplete data CanLoad Prevent load restricted lazy loading module Resolve Load data before navigating

Executed in this order. Lets say you are navigating from /home to /outside/shop

1. CanDeactivatewhen leaving /home route 2. CanLoadbefore loading lazy module 3. CanActivateChildin parent /outside route 4. CanActivatein /outside/shop child route 5. Resolvein /outside/shop child route
